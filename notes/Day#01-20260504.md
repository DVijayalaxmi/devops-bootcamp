## Create AWS EC2 Instance

### 1. Navigate to the EC2 Dashboard
- Sign in to your AWS account.
- Search for "EC2" in the top search bar or find it under the "Compute" section in the services menu.
- Select the AWS Region where you want your server located (e.g., US East) from the top-right corner.
### 2. Launch Instance
- On the EC2 Dashboard, click the Launch Instance button.
- Name and Tags: Enter a name for your instance (e.g., "MyWebServer") to help identify it later.
### 3. Choose an OS (AMI)
- Select an Amazon Machine Image (AMI), which is the operating system for your virtual machine.
- Common choices include Amazon Linux 2023 or Ubuntu Server, both of which are Free Tier eligible.
### 4. Select Instance Type
- Choose the hardware configuration (CPU, RAM). For testing and learning, select the t2.micro (or t3.micro, depending on your region), which offers 1 vCPU and 1 GB of RAM for free.
### 5. Configure Key Pair
- Create a new key pair or select an existing one to securely log into your server via SSH.
- **Important:** Download the private key file (.pem or .ppk) and store it safely; you cannot download it again after this step.
### 6. Network and Security Settings
- **Security Group:** Create a virtual firewall rule.
- Ensure Allow SSH traffic is checked so you can connect.
- If you plan to host a website, check Allow HTTP/HTTPS traffic from the internet.
### 7. Configure Storage
- The default is typically 8 GB (SSD). The AWS Free Tier allows up to 30 GB of total EBS storage.
### 8. Review and Launch
- Review your settings in the summary panel and click Launch Instance.
- It will take a few minutes for the instance status to change from "Pending" to "Running".

## Docker

#### Docker: Accelerated Container Application Development

Docker is a platform designed to help developers build, share, and run container applications. Docker handles the tedious setup, so we can focus on the code.


### Docker Installation on Ubuntu

#### Run below commands to install docker on ubuntu.

*Run the following command to uninstall all conflicting packages:*

```
sudo apt remove $(dpkg --get-selections docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)
```

*Set up Docker's apt repository.*

```
# Add Docker's official GPG key:
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update
```

*Install the latest Docker packages.*

```
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

*Install the specific docker version*

```
apt list --all-versions docker-ce

VERSION_STRING=5:29.4.2-1~ubuntu.24.04~noble
sudo apt install docker-ce=$VERSION_STRING docker-ce-cli=$VERSION_STRING containerd.io docker-buildx-plugin docker-compose-plugin
```

*After installation, verify that Docker is running:*

```
sudo systemctl status docker
```

*If Docker is not running, start it manually:*

```
sudo systemctl start docker
```

*Verify that the installation is successful by running the hello-world image:*

```
sudo docker run hello-world
```
